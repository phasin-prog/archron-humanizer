# Navigation Model

Users can traverse knowledge in multiple ways.

## Navigation Modes

### 1. Search
Direct query → results. The primary entry point for goal-oriented users.

### 2. Browse
Topical exploration by Domain → School → Theory → Concept. For users who want to learn a subject area systematically.

### 3. Timeline
Chronological exploration. Events, publications, and births/deaths arranged along a timeline. For understanding intellectual history.

### 4. Knowledge Graph
Visual exploration of connections. See how Thinkers, Concepts, and Books relate in a network visualization.

### 5. Related Content
Contextual recommendations. Every Object page shows related Objects based on shared relationships, domains, or tags.

### 6. Collections
Curated pathways. User-created or editor-curated groups of Objects around a theme or learning goal.

### 7. Guides
Structured learning journeys. Sequential lessons that reference Objects in a defined order.

## Navigation Principles

- Every screen must offer at least 2 navigation modes
- Search must always be accessible (header or global shortcut)
- Breadcrumbs show the taxonomic path: Discipline → School → Theory
- Backlinks show which Objects reference the current Object
- No more than 3 clicks to reach any Object from the homepage
