# Design Philosophy

## The Hierarchy

```
Information First
    │
    ▼
Interaction Second
    │
    ▼
Animation Third
```

**Information** always wins over **Animation**. Every time.

## Principles

### Information Before Decoration
Before adding a decorative element, ask: does this help understanding? If not, remove it.

### Knowledge Before Technology
Choose the typography that best presents the content, not the trendiest font. Choose the layout that best organizes information, not the most innovative grid.

### Relationship Before Hierarchy
A Concept page should not just be a heading + text. It should show how this Concept relates to other Concepts, Thinkers, and Books. Relationships are content.

### Objects Before Pages
Design for the Object, not the screen. Every Concept should feel like it has a home, not a page. The layout serves the Object type.

### Consistency Before Novelty
Every Concept page looks like every other Concept page. Every Thinker page follows the same structure. Predictability is a feature.

### Fast Before Fancy
A fast-loading page with minimal design beats a slow-loading page with animations every time.

### Minimal Before Crowded
When in doubt, remove. Every element must earn its place on screen.

### Premium Through Clarity
Premium design is not gold gradients or fancy textures. It is sharp typography, generous spacing, and perfect alignment.

## Visual Tone

- Quiet, not loud
- Calm, not exciting
- Clear, not clever
- Warm, not cold
- Academic, not institutional
