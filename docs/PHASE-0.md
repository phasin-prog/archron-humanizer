# ARCHRON — Phase 0
## Project Constitution

Version: 0.1
Status: Foundation
Purpose: This document defines the philosophy, boundaries, architecture direction, and design principles of the ARCHRON platform before implementation begins.

---

# 1. Vision

ARCHRON is not a blog.

ARCHRON is not a CMS.

ARCHRON is not an encyclopedia.

ARCHRON is a Human Knowledge Platform designed to organize, connect, and transform multidisciplinary knowledge about human beings into an interconnected knowledge ecosystem.

Knowledge is treated as a living network rather than isolated pages.

---

# 2. Mission

Reduce the time between

Question
↓

Understanding

Every design decision should reduce cognitive friction.

---

# 3. Core Philosophy

Information before decoration.

Knowledge before technology.

Relationship before hierarchy.

Objects before pages.

Consistency before novelty.

Fast before fancy.

Minimal before crowded.

Premium through clarity.

# 8. Rendering Philosophy

Markdown is the source.

Renderer is the brain.

React Components are the presentation.

---

# 9. Platform Philosophy

Studio manages knowledge.

Editor creates knowledge.

Renderer transforms knowledge.

Knowledge Graph connects knowledge.

Search discovers knowledge.

---

# 10. UX Philosophy

The user should understand where they are within three seconds.

The interface should disappear.

Typography is more important than decoration.

Whitespace is part of the content.

Motion should never compete with information.

# 14. Anti-Goals

Do not become another WordPress blog.

Do not become another Medium clone.

Do not become another Wikipedia clone.

Do not overload the interface.

Do not sacrifice readability for visual effects.

Do not prioritize animations over information.

---

# 15. Development Rule

Every feature must answer

Why does it exist?

Who benefits?

Where does it belong?

How does it connect?

If these questions cannot be answered,
the feature should not be implemented.
