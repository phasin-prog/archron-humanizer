# Roles & Responsibilities

A Role is not a badge. A Role is a **responsibility**.

## Role Hierarchy

```
                        Guest
                           │
                        Member
                           │
                       Supporter
                           │
                      Contributor
                       /       \
                   Writer    Reviewer
                       \       /
                        Editor
                           │
                        Curator
                           │
                     Administrator
```

## Role Definitions

### Guest
- **Access**: Read published content, search, browse
- **Limitations**: Cannot save, create, or personalize
- **Purpose**: First interaction with the platform

### Member
- **Access**: Everything Guest has + create Collections + reading progress
- **Requirements**: Email registration
- **Purpose**: Personal knowledge management

### Supporter
- **Access**: Everything Member has (no content difference)
- **Requirements**: Coffee contribution
- **Purpose**: Sustain the platform financially — gratitude, not privilege

### Contributor
- **Access**: Everything Supporter has + submit corrections + suggest references
- **Requirements**: Account age ≥ 30 days + accepted Contributor Agreement
- **Purpose**: Improve knowledge quality through small contributions

### Writer
- **Access**: Everything Contributor has + create Drafts + upload Media
- **Requirements**: 5 accepted corrections + domain test + trial period
- **Purpose**: Create original knowledge content

### Reviewer
- **Access**: Everything Writer has + review Drafts + validate content
- **Requirements**: Published 5+ Articles + domain expertise verified by Editor
- **Purpose**: Ensure knowledge accuracy and quality

### Editor
- **Access**: Everything Reviewer has + approve/publish/archive content + manage writers
- **Requirements**: Nominated by existing Editor + Administrator approval
- **Purpose**: Maintain knowledge quality and guide content direction

### Curator
- **Access**: Everything Editor has + create featured collections + manage guides
- **Requirements**: Appointed by Editor team
- **Purpose**: Shape the learning experience and highlight quality content

### Administrator
- **Access**: Full system access + user management + configuration
- **Requirements**: Owner appointment
- **Purpose**: Platform operations and governance

## Role Assignment Rules

- Roles are earned, not requested (except Member)
- Role downgrade possible for inactivity or quality issues
- Trial period for new Writers: 30 days with mentor
- Role changes are logged in the audit trail
- No role can self-assign permissions
- Role-based access is enforced at the Knowledge Engine level
